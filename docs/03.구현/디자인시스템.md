# 디자인 시스템 — Discord 기반 (임시 확정)

> 디자인 시스템의 출처는 저장소 루트의 [DESIGN.md](../../DESIGN.md)입니다. Discord의 디자인 언어를 분석한 토큰·컴포넌트 명세이며, 이 프로젝트는 이를 그대로 채택합니다.
> 이 문서는 그 명세를 Angular + Tailwind v4 구조에 어떻게 녹일지를 정합니다.

## 출처와 성격 — 한 가지 짚을 점

DESIGN.md는 Discord의 **마케팅 페이지** 디자인 언어입니다. deep-indigo 캔버스 위의 Blurple, 82px에 이르는 all-caps 디스플레이 타입, 둥근 기하가 특징입니다. 이 언어는 랜딩 페이지에서 "크게 외치도록" 설계되었습니다.

우리가 만드는 것은 모바일 우선 앱입니다. 따라서 토큰(색·간격·라운드)과 컴포넌트 크롬은 그대로 채택하되, **마케팅 스케일의 타이포(82~56px)는 앱 UI에 그대로 쓰지 않고 조정**합니다. 이 차이는 뒤의 "조정 지점"에 정리합니다. 토큰을 버리는 것이 아니라, 같은 토큰을 앱 맥락에 맞는 크기에서 쓰는 것입니다.

## 3계층 — 디자인을 소유하는 방식

1. **토큰** — DESIGN.md의 값을 Tailwind `@theme`에 정의합니다.
2. **프리미티브** — `shared/ui`가 CDK + Tailwind로 토큰을 참조해 만듭니다.
3. **컴포넌트** — `entities`·`features`가 프리미티브를 조합합니다.

원칙은 변하지 않습니다. **Tailwind 클래스는 구현 디테일이며, 공개 API가 아닙니다.** 디자인 시스템은 클래스 이름이 아니라 컴포넌트를 노출합니다.

## 토큰 매핑 (Tailwind v4 `@theme`)

Tailwind v4는 설정을 CSS에서 합니다. `src/styles.css`의 `@theme` 블록에 DESIGN.md 값을 옮기면, 대응하는 유틸리티(`bg-primary`, `rounded-xl`, `p-md`, `tablet:flex`)가 생성됩니다.

```css
@import "tailwindcss";

@theme {
  /* 색 — DESIGN.md colors */
  --color-primary: #5865f2;        /* Blurple */
  --color-green: #35ed7e;          /* 최고 강조 CTA 전용 */
  --color-magenta: #ec48bd;
  --color-link: #00b0f4;
  --color-canvas: #0a0d3a;
  --color-surface-indigo: #1e2353;
  --color-surface-onyx: #23272a;
  --color-ink: #ffffff;

  /* 라운드 */
  --radius-xs: 6px;  --radius-sm: 12px; --radius-md: 14px;
  --radius-lg: 16px; --radius-xl: 40px; --radius-pill: 50px;

  /* 간격 (8px 베이스) */
  --spacing-xxs: 4px; --spacing-xs: 8px;  --spacing-sm: 12px;
  --spacing-md: 16px; --spacing-lg: 20px; --spacing-xl: 24px;
  --spacing-xxl: 32px; --spacing-section: 40px;

  /* 브레이크포인트 — DESIGN.md 사다리 */
  --breakpoint-tablet: 768px;
  --breakpoint-laptop: 1024px;
  --breakpoint-desktop: 1280px;

  /* 폰트 (아래 "폰트" 참고) */
  --font-display: "Space Grotesk", system-ui, sans-serif;
  --font-body: "Inter", system-ui, sans-serif;
}
```

## 폰트 — 독점 → 오픈소스 대체

DESIGN.md의 ABC Ginto Nord·ggsans는 독점 폰트입니다. DESIGN.md 자체 권고를 따라 대체합니다.

- **디스플레이**: Space Grotesk 또는 Hanken Grotesk (700–800)
- **본문·UI**: Inter 또는 Plus Jakarta Sans

무게 점프(800 → 400)가 이 시스템의 목소리이므로, 헤드라인은 굵게 유지합니다.

## 프리미티브 ↔ DESIGN.md 매핑

`shared/ui` 프리미티브가 DESIGN.md의 어느 컴포넌트에서 크롬을 가져오는지 정리합니다.

| 프리미티브 | DESIGN.md 출처 |
| ---------- | -------------- |
| Button (기본) | `button-primary` (Blurple, `rounded-sm`) |
| Button (CTA) | `button-green` (최고 강조, 검은 글자) |
| Button (ghost) | `button-ghost` (`surface-indigo`) |
| Sheet · Dialog | `ex-modal-card` (`canvas`, `rounded-lg`, 그림자) |
| Snackbar | `ex-toast` |
| EmptyState | `ex-empty-state-card` |
| Badge · Chip | `badge` (`magenta`) |
| ListItem (할 일 항목) | `game-rank-row` (`surface-indigo`, `rounded-md`) |
| Nav 활성 표시 | `ex-app-shell-row` (활성 = `primary`) |
| Card | `feature-card-dark` |
| 인증 폼 표면 | `ex-auth-form-card` |

디스코드의 랭킹 행(`game-rank-row`) 크롬이 우리 "할 일 항목" 행에 거의 그대로 들어맞습니다. 마케팅 컴포넌트가 앱 프리미티브로 재사용되는 지점입니다.

## 조정 지점 — 마케팅에서 앱으로

디스코드 시스템을 앱에 녹일 때 다듬는 부분입니다.

- **타이포 스케일** — 82~56px 디스플레이는 마케팅용입니다. 앱의 본문·헤더는 `heading-sm`(22) · `body-lg`(20) · `body`(16)를 중심으로 쓰고, 디스플레이는 스플래시·빈 화면·온보딩 같은 "큰 순간"에만 씁니다.
- **색의 절제** — `green`은 화면당 최고 강조 한 곳에만 씁니다(DESIGN.md 규칙). 앱에서는 주요 행동(예: 할 일 추가)에 한정합니다. `magenta`는 강조 패널과 배지에 씁니다.
- **그림자보다 색·라운드** — 깊이는 그림자가 아니라 색·라운드·겹침으로 만듭니다. 떠 있는 표면(시트·토스트)에만 DESIGN.md의 단일 violet glow를 씁니다.
- **모션 그라디언트** — Blurple→magenta 메시는 분위기 시그니처입니다. 앱에서는 스플래시·헤더 등 제한된 곳에만 씁니다.

## 반응형 전략 — 모바일 우선, 이후 웹

> 질문: "모바일 first이지만 이후 웹 디자인도 챙긴다 — 어떻게?" → 모바일을 기준으로 짜고 위로 확장합니다. 두 층위로 다룹니다.

**1층위 — 유틸리티 반응형 (같은 레이아웃, 크기·간격만 조정).**
Tailwind는 모바일 우선입니다. 접두 없는 클래스가 모바일 기본이고, `tablet:` · `laptop:` · `desktop:` 접두로 위 화면을 덧입힙니다. 브레이크포인트는 위 `@theme`의 DESIGN.md 사다리를 씁니다.

| 이름 | 폭 |
| ---- | --- |
| mobile | < 768px |
| tablet | 768–1023px |
| laptop | 1024–1279px |
| desktop | ≥ 1280px |

**2층위 — 구조적 적응 (레이아웃 자체가 바뀜).**
크기 조정으로 안 되는 전환은 `BreakpointObserver`로 분기합니다(스토리보드의 적응형 규칙과 동일).

- 할 일 추가·편집: 바텀시트(모바일) ↔ 모달(데스크톱)
- 내비게이션: 하단 탭(모바일) ↔ 사이드 내비(데스크톱)

같은 컴포넌트가 표현만 바꿉니다. 이 분기는 `shared/ui`의 `Sheet`·`Nav` 안에 캡슐화해, 화면 코드가 분기를 신경 쓰지 않게 합니다. 터치 타깃은 모바일 기준 최소 44px를 지킵니다(DESIGN.md).

## 다크 · 라이트

디스코드 시스템은 deep-indigo 다크 캔버스를 기반으로 합니다. 설정에 테마 전환이 있으므로, **다크를 기본**으로 두고 라이트는 폴라리티 플립(밝은 면 + 어두운 글자)으로 둡니다. 토큰을 CSS 변수로 스왑하는 방식(`:root` ↔ `[data-theme="light"]`)을 쓰되, 구체 라이트 팔레트는 설계 단계에서 확정합니다(미확정).
