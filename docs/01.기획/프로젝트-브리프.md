# 프로젝트 브리프 — 투두리스트 플랫폼 (데모)

> 확정된 사항을 기록하는 문서입니다. 검토 과정과 대안은 [리서치](%EB%A6%AC%EC%84%9C%EC%B9%98.md)를 참고하세요.
> 화면·데이터 모델 등 구체적인 설계는 기능별 설계 문서에서 다룹니다(추후 작성).

## 무엇을 만드나

투두리스트 플랫폼을 만듭니다. 다만 목적은 제품 자체가 아니라, 그 위에서 **모바일 UX 패턴을 구현하고 검증**하는 데 있습니다. 투두리스트는 누구에게나 친숙하고 상호작용이 풍부해, 바텀시트·제스처·알림 같은 패턴을 얹어 보기에 적합한 무대입니다.

## 범위 (기능)

초기 범위는 네 영역입니다. 각 영역의 세부 사항은 설계 단계에서 확정합니다.

| 영역 | 내용 |
|------|------|
| 투두리스트 | 할 일 생성 · 편집 · 완료 · 정렬 |
| 사용자 | 프로필, 인증 (방식은 설계 시 결정) |
| 알림 | 리마인더, 알림 센터 |
| 설정 | 테마, 알림 설정 등 |

## 확정 기술 스택

선택 근거는 [리서치 4절](%EB%A6%AC%EC%84%9C%EC%B9%98.md#4-%EC%9D%B4-%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B8%EC%97%90-%EC%96%B4%EC%9A%B8%EB%A6%AC%EB%8A%94-%EA%B8%B0%EC%88%A0-%EC%8A%A4%ED%83%9D)을 참고하세요.

| 항목 | 결정 |
|------|------|
| 프레임워크 | Angular 22 (CLI 22.0.3) |
| UI 토대 | Angular CDK 단독 |
| 스타일 | Tailwind CSS v4 |
| 렌더링 | CSR (클라이언트 전용) |
| 앱 형태 | PWA (service worker + manifest) |
| 데이터 저장 | 프론트엔드 영속화 — IndexedDB(Dexie.js) + localStorage |
| 테스트 | Vitest |

## 범위 밖

- **SSR · 라우트별 혼합 렌더링** — 이 데모의 화면은 대부분 브라우저 런타임에 의존해 SSR의 이점이 작습니다. 그 자체로 깊은 주제이므로 별도 프로젝트에서 다룹니다.
- **백엔드(서버)** — 서버는 두지 않습니다. 데이터는 위 표대로 브라우저에 영속화합니다. 비교와 근거는 [데이터 저장 리서치](%EB%8D%B0%EC%9D%B4%ED%84%B0-%EC%A0%80%EC%9E%A5-%EB%A6%AC%EC%84%9C%EC%B9%98.md)를 참고하세요. 원격 동기·서버 저장은 범위 밖이며, 저장소 접근을 서비스로 추상화해 이후 도입 여지만 남깁니다.

## 다음 단계
